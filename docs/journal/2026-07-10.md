## 19:03 — FastV baseline sh0+sh1 完成！结果出乎意料

### FastV L2 r=0.5 PDMS = **0.831**（比 random 还低！）

| Selector @ r=0.5 | PDMS | Δ vs no-prune | Δ vs scorer |
|---|---|---|---|
| **Scorer (LambdaRank)** | **0.8920** | −0.69pt | — |
| Attention L12 (teacher) | 0.8902 | −0.87pt | −0.18pt |
| Random | 0.8636 | −3.52pt | −2.84pt |
| **FastV L2 (layer-2 attn)** | **0.8314** | **−6.74pt** | **−6.06pt** |

**FastV L2 不仅比 scorer 差，甚至比 random 还差 3.2pt！**

### 解读（非常重要）

1. **Layer-2 attention 的信息质量极差**：LLM 前两层的 attention 几乎没有学到 task-relevant 的 token 重要性
2. **这直接证明 "pruning position" 不是关键，"selector quality" 才是**：
   - FastDriveVLA 方法论 = shallow attention as selector（虽然它在 LLM 内部剪，不在入口）
   - 我们证明：即使在同一位置（入口）剪，shallow attention 依然不行
3. **Scorer 的 gain 不是来自"在入口剪而非内部剪"，而是来自 learned ranking quality**
4. **论文叙事增强**：C1 差异化地基比预期还强——scorer 不仅超 teacher(+0.18pt)，还碾压 FastV 方法论(+6.06pt)

### 对竞品 FastDriveVLA 的定位

FastDriveVLA 用 MAE 重建学前景 mask → 本质是一种 learned selector（但基于 pixel-level 重建而非 task-level ranking）。我们的 FastV-at-input baseline 是其方法论的"最简实例化"（用 raw attention 代替 MAE）。实际的 FastDriveVLA 会比我们的 FastV baseline 好，但我们的论点是：**排序蒸馏式 selector 是原则性更优的方法**。
