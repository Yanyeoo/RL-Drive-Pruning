## 一、项目是什么

**RL-Drive**：面向自动驾驶 VLA 的**驾驶场景条件化视觉 token 剪枝**，用 RL 学习。
- **动机**：VLA 的 FLOPs 绝大部分花在 vision token 的 LLM prefill（~70%），但很多 token 对当前驾驶决策冗余。
- **骨干**：AutoVLA（Qwen2.5-VL-3B），frozen。ckpt `models/AutoVLA/AutoVLA_PDMS_89.ckpt`。
- **Benchmark**：NAVSIM **navtest** open-loop PDMS/EPDMS（06-15 从 navhard_two_stage 切换，因 AutoVLA 上游 navsim fork 不原生支持 navhard 双路径）。
- **三条核心 claim（待验证）**：①≥50% token 节省几乎不掉点；②自适应 budget > 固定比例；③驾驶条件化 scorer ≫ 纯 vision-attention。

## 二、设计冻结（2026-06-14，Q1–Q5）

**两阶段架构**（frozen 骨干；剪枝在 ViT→LLM 接口 = external pruning）：

| 阶段 | 模块 | 目标 | 监督 |
|---|---|---|---|
| **Stage A** | Importance Scorer（轻量 MLP）| token 按驾驶相关性排序 | LLM-attention 蒸馏(LambdaRank) → GRPO(EPDMS-advantage) |
| **Stage B** | Budget Policy | 每场景选 keep 比例 {0.25/0.5/0.75/1.0} | B3 Pareto-aware oracle(SFT) + R3 piecewise Pareto reward(GRPO) |

关键决策：G4-with-camera-id（budget 跨相机自由流通）；scene-context 三件套(nav_cmd+ego_speed+driving_instr)；两阶段 separate；per-scene advantage 降方差；reward clip[-1,1]。评测：7 baselines + A1–A10 ablations，主报 navtest ratio=0.5 + Pareto 曲线。
