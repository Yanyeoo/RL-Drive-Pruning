# `docs/reports/` — milestone reports

One markdown report per completed milestone (M0–M6), capturing:

- Setup (config, data split, hardware).
- Headline numbers (EPDMS, FLOPs, latency, kept-ratio).
- Plots / tables (or links to `results/` artifacts).
- Notable findings, surprises, anomalies.
- Decisions / next steps that arose from the milestone.

Naming:
```
docs/reports/M<id>_<short-tag>.md          e.g.  M0_baseline.md
                                                 M2_stageA_sft.md
                                                 M5_grpo_full.md
```
