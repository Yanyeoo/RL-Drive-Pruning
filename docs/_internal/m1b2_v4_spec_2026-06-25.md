## 1. Motivation

M1.b₁ V3 (`L12:{h13} + L24:{h0,1,2,6,7,8,9,10,12,14,15} + L27:{h0,8,9}`) on navtest scored PDMS = 89.75 vs V2's 89.83, a tiny −0.08 pp gap that masked **per-scene heterogeneity** inside the L24 mask.

The rank-variance analysis (2026-06-25 18:23–18:25, all 19,225 navtrain tokens) revealed that **only 3 of the 11 heads in L24 are reliably bot-K per scene**:

| L24 head | bot-4 freq (navtrain) | Verdict |
|---:|---:|---|
| h7  | 93.67% | ✅ reliably dead |
| h9  | 99.98% | ✅ reliably dead |
| h10 | 98.92% | ✅ reliably dead |
| h0  | 53.51% | ⚠ borderline |
| h12 | 16.60% | ❌ mostly NOT weak |
| h15 | 19.79% | ❌ mostly NOT weak |
| h6  | 13.04% | ❌ mostly NOT weak |
| h14 | 4.46%  | ❌ almost never weak |
| h1  | 0.03%  | ❌ never weak |
| h2  | 0.00%  | ❌ never weak (rank_mean 9.83) |
| h8  | 0.00%  | ❌ never weak (rank_mean 8.96) |

V3's −0.08 pp on navtest is "compensation by redundancy": masking 8 non-dead heads cancels out against the 3 truly dead heads. On navtrain (out-of-distribution scenes) we predict V3 will break down (R3 hypothesis).

**V4 keeps only the 3 reliably-dead L24 heads**, giving a principled "free-lunch L24 add-on" rather than V3's blanket 11-head mask.

## 2. Specification

### 2.1 Mask definition

```python
head_mask_layers = {
    12: [13],          # V1 mask — L12:h13 always rank-0 (free-lunch on navtest & navtrain)
    24: [7, 9, 10],    # V4 add — only the 3 reliably bot-4 heads on navtrain
}
```

Hydra string form (for `+agent.head_mask_layers=...`):

```
'{12: [13], 24: [7, 9, 10]}'
```

### 2.2 What V4 IS NOT

- ❌ NOT V3 (`L24:{0,1,2,6,7,8,9,10,12,14,15}`) — 8 of those 11 heads are not reliably dead.
- ❌ NOT V2 (`L12:{h13} + L27:{0,8,9}`) — V4 deliberately skips L27 to isolate the L24 ablation.
- ❌ NOT learned (this is still a *static* mask; learned per-scene policy = M1.b₂ Phase 2, separate effort).

### 2.3 Total masked heads

- V0: 0 heads
- V1: 1 head (L12:h13)
- V2: 4 heads (L12:h13 + L27:{0,8,9})
- V3: 15 heads (L12:h13 + L24:{0,1,2,6,7,8,9,10,12,14,15} + L27:{0,8,9})
- **V4: 4 heads (L12:h13 + L24:{7,9,10})**

Same head count as V2, different layer placement (L24 vs L27).

## 3. Hypotheses to test

| H# | Hypothesis | Prediction |
|---:|---|---|
| H1 | V4 matches V1 on navtest (no regression from added L24 heads). | PDMS(V4 navtest) ≥ V1 PDMS − 0.1 pp |
| H2 | V4 beats V3 on navtrain — rank-variance theory predicts V3 brittleness from 8 non-dead heads. | PDMS(V4 navtrain) > PDMS(V3 navtrain) by ≥ 0.3 pp |
| H3 | V4 = V2 on navtest in numerical-rank terms (same head count, different layer). | Sub-metric breakdown shows L24 mask trades nothing vs L27 mask. |

H2 is the key contribution; if it holds, the per-scene rank-variance analysis directly improves M1.b₁ over its published static masks.

## 4. Evaluation plan

### 4.3 Acceptance gates

| Gate | Target | Required for |
|---|---|---|

## 6. Risk register

| R# | Risk | Mitigation |
|---|---|---|
| R1 | L24:h7/9/10 rank-mean is computed on navtrain, evaluated on navtest → distribution shift hides cliff. | C1 dryrun on navtrain_probe100 is the canary; if PDMS << B0 we re-examine before launching full navtrain. |
| R2 | "bot-4 freq" cutoff (93/99/98%) is arbitrary; h0 at 53.51% might also be safe. | Reserve V4b = `+L24:{0}` as a follow-up variant pending V4 result. |
| R3 | navsim's 8-trajectory-assert denylist excludes scenes from PDMS but not from cache → eval may inflate n_total. | Filter denylist in `merged.csv` post-aggregation, document n_excluded in manifest. |
