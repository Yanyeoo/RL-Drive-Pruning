### 结论 (改变 Step2 实现): dynamic mask 粒度 = **per-scene**, 不是 per-(generated)-token

- feature_96d 是**每个 scene 一个** 96-d 向量 (对 720 vision token mean over kv 后, 6 层 × 16 head).

## 偏离 #2 — dynamic = per-scene; head_mask_patch 不需 per_token patch (改 step2_spec)

**step2_spec §1 + §6 checklist 写**: `patch_head_mask({12: to_mask_heads}, per_token=True)`, mask shape (T,16), 并把 "head_mask_patch per_token mode" 列为 🔴 P0.

**事实**: R1pp dataset 的 token = scene (上面核验), feature/label 都是 per-scene. **per-(generated)-token 粒度在当前 dataset 下没有任何训练数据支持** — 要 per-token 必须重建一个 per-decode-token feature dataset (现在没有, 且不在 design doc).

**决定** (按 dataset 硬事实纠正 spec 笔误, 非主观选择):
1. Step 2 MVP = **per-scene** dynamic mask.
2. `head_mask_patch.py` **不加 per_token patch** (省一个 P0). per-scene 用现有 constant-mask, 每 scene 新 context.
3. `autovla_with_dynamic_mask.py` 实现 per-scene 2-pass: pass1 prefill capture L0/4/8/16/20/24 mean attn → 96-d → probe → drop_heads → pass2 `patch_head_mask({12: drop_heads})` 跑 generate.
4. per-token (per decode token) 标为 **future / Phase 4** 方向, 需先建 per-token feature dataset.

**Reverse 指令**: 若 user 坚持要 per-token: (a) 需新建 per-decode-token feature dataset (capture 每个 decode step 的 6 层 attn); (b) head_mask_patch 加 per_token=True (mask shape (q_len,16), hook 里 `x[:,t,h*hd:(h+1)*hd]=0 where mask[t,h]==0`); (c) 我保留了 `head_mask_patch.py.bak_20260630_premask` 备份, 原文件未动.

## 偏离 #3 — task_loss="real" 在离线 dataset 上不可行 (记录, 不阻塞 MVP)

**spec**: train_probe.py 的 `--task_loss real` 接通真 logit KL.

**事实**: R1pp dataset 只有 96-d feature + label, **无原始 scene 输入** (images/video). 要跑 backbone 拿 action logits 必须有 scene → 离线 dataset 训不了 real task loss. real task loss 需 online (在 navtrain scene 上一边 generate 一边训), 是另一套 data pipeline.

**决定**:
- Step 2 MVP 训练用 `--task_loss surrogate_kl` (BCE vs 1-bot_K=4, 已通) 或 `none` (纯 sparsity), 在离线 dataset 上训出 probe.
- **eval 才是关键**: 把训好的 probe 装进 `autovla_with_dynamic_mask.py` 跑 navtest, 直接量 dynamic (avg_K_eff, PDMS), 与 static 曲线比 Pareto. eval 不需要 task_loss real.
- task_loss real 留作后续 (online 训练) 或干脆不做 (surrogate 训出的 probe 若 eval Pareto dominate 就够了, per user "只要效果好").

## 16:35 — 偏离 #4: surrogate_kl BCE 梯度 bugfix (修改 _drafts 训练脚本)

**现象**: λ-sweep 训出的 12 个 probe **全部 eval K_eff=0**; CPU 诊断进一步发现 surrogate_kl 在 lr 跨 300× / epoch 30~80 / λ 跨 300× 下 **val_task 恒 = ln(2)=0.6931、train_K_eff 恒 = 8.00**, 即 probe 参数从未更新.

**根因 (autograd bug)**: 训练 loop `mask = probe(x, hard=True)` 产出 STE-hard mask (值恰好 0.0/1.0), 喂给 `surrogate_kl_loss` 内 `m = mask.float().clamp_(1e-6, 1-1e-6)`. **所有 hard 值都落在 clamp 边界外 → clamp 局部梯度=0 → 16 个 head 全部梯度被清零 → W/b 永冻初始值**. 这一个 bug 同时解释了:
- val_task 恒 ln(2) (logits 冻在 0, keep_prob=0.5)
- train_K_eff 恒 8 (纯 gumbel 随机一半)
- 最初 12 个 probe 全 eval K_eff=0 (bias=2.0 时同样梯度被杀, 永停 keep-all)

**修复** (`scripts/_drafts/m1b2_phase3_step2_train_probe.py` train loop, 非锁定文件; 备份 `*.bak_20260630_bcefix`): surrogate 分支改喂 **soft keep_prob** (`torch.sigmoid(probe.net(x))`) 而非 STE-hard. BCE 本就需要概率而非 0/1.
- **reverse**: `cp -a scripts/_drafts/m1b2_phase3_step2_train_probe.py.bak_20260630_bcefix scripts/_drafts/m1b2_phase3_step2_train_probe.py`

**验证** (CPU, /tmp/p3s2_diag3): bias=2.0/lr=3e-3/ep=30/λ=0 → **val_K_eff=4.00** (=删 bot-4), val_task 0.69→0.19. 梯度通了, probe 真在学.

**同时确认天花板 (对应偏离 #3)**: surrogate+sparsity λ 0→0.5, **val_K_eff 恒 4.00** (train 软态 3.94→4.46, eval 阈值化后不动). surrogate 把 probe 死锚在"删 bot-4"≈**静态 Sc4**, per-scene 自适应空间极小, eval 阈值下推不出 K_eff>4. ⇒ offline surrogate 只能复现 Sc4, 不可能 Pareto dominate; 真 dominate 仍需 task_loss=real (online, 偏离 #3).

**今晚定位 (GPU deadline 24:00, 窗口 ~17:50–24:00)**: A 路 = 训 1 个 surrogate probe (K_eff≈4) → GPU 空后 dynamic eval 端到端验通 pipeline + 看 per-scene 自适应能否略胜 static Sc4(0.8984). 不跑 12 个 (其余 λ 在 eval 下都=4, 浪费卡).
