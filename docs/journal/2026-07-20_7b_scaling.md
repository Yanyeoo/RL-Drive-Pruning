## 2. 目标 & 策略

**C2 (Scaling Law)**: 验证"7B 模型比 3B 更耐剪"——即 7B 在 r=0.50 时 PDMS drop 相对于 3B 更小。

**策略**: 用 7B base model (无 driving fine-tune) 做 feature dump + attention probe → 训练 7B scorer → 评测 7B token pruning tolerance。

**关键 insight**: 即使 7B base 的绝对 PDMS 低（无 driving fine-tune），**相对剪枝容忍度** (r=0.75/0.50 vs r=1.0 的 delta) 仍能支撑 scaling law。

## 3. Code 修复 (规则 #4: 偏离原 design 的决策)

### 3.1 允许空 checkpoint 路径
- **文件**: `code/rldrive/scoring/run_feature_dump.py`, `code/rldrive/scoring/run_attention_probe.py`
- **改动**: `if pth and not Path(pth).exists()` (原: `if not Path(pth).exists()`)
- **理由**: 7B base model 无 fine-tuned ckpt，传空字符串应合法
- **Reverse**: 恢复 `if not Path(pth).exists():`

### 3.2 添加 7B config 的 inference section
- **文件**: `code/third_party/AutoVLA/config/training/qwen2.5-vl-7B-navtest-grpo-nocot.yaml`
- **改动**: 添加 `inference: {batch_size: 1, num_workers: 1, sample: {max_length: 2048, temperature: 0.01, top_k: 0.0, top_p: 1.0}}`
- **理由**: AutoVLA 初始化需要 `config['inference']['sample']`，原 7B config 缺失导致 KeyError
- **Reverse**: 删除 inference 段落

### 3.3 捕获 predict 错误并保存 features
- **文件**: `code/rldrive/agents/autovla_with_attention.py`
- **改动**: wrap `self.autovla.predict(features)` in try/except，在 capture-only 模式下返回 dummy trajectory
- **理由**: 7B base model 不能正确 decode action tokens（未 driving fine-tune），但 feature capture hooks 在 prefill 阶段已触发，需保存
- **Reverse**: 移除 try/except，恢复原始 `poses, cot_results = self.autovla.predict(features)`

### 3.4 修复 run_7b_pipeline.sh 的 --multi-layer → --all-layers
- **文件**: `scripts/run_7b_pipeline.sh`
- **改动**: `--multi-layer` → `--all-layers --num-layers 28 --per-head`
- **理由**: argparse 定义的是 `--all-layers` 不是 `--multi-layer`
- **Reverse**: 无需，纯 bugfix
