## 3. GRPO Scorer RL — 核心方向决策 ⚠️

### 偏离 #1: 不跑 VLM GRPO，改为直接训 Scorer RL

**原因**: 
- 论文 story = "First RL-optimized **token pruning** for AD-VLA"
- VLM GRPO (原有代码) 训的是 VLM 本身，不涉及 token selection
- Scorer RL 直接用 REINFORCE + PDMS reward 优化 token selection = 论文核心贡献
- TOP-RL [AAAI 2026] 已做了 "RL for token pruning" 在 VLM/VQA 上 → 我们必须做等价的在 AD 上

**Reverse**: 如果 Scorer RL 失败（reward 不上升），回退到 VLM GRPO → 蒸馏新 attention → 重训 scorer

**设计**:
```
Per scene:
  1. VLM pass-1 → capture vision features (frozen)
  2. Scorer(features) → scores [HAS GRAD]
  3. Top-B selection → prune mask
  4. VLM pass-2 → generate trajectory (frozen)
  5. PDMS reward
  6. REINFORCE: loss = -advantage * log_prob(selection | scores)
```

### Smoke Test 结果 ✅ (14:55)

- 代码: `scripts/train_scorer_grpo.py`
- 20 scenes, 5 groups × 4 scenes, 1 epoch
- **全流程通过**: reward 正常(0.68-0.96), 梯度传播正常, ckpt 保存正常
- 速度: ~4.4 s/scene (单卡 H20)
- 发现: log_prob 数值过大 (B=360 时) → 已修复(除以 B 归一化)
