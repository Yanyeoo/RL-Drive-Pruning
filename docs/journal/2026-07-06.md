## 2. 本周期目标（价值优先，bounded）

> **背景约束**：HANDOFF_2026-07-04 明确「建 S3 前先与 user 对齐范围（多天工程），勿盲目开工」。故本周期**只做 S3 里 bounded、不需多天、不碰 online-GRPO 的那一片** = **M1c：token Importance Scorer 的 SFT + 作为 selector 的离线评测**。

**主目标（Track B）：训练 per-token Importance Scorer 并作为 selector 评测。**
- 设计（design Q3.a/Q4.1）：input = [ViT→LLM 接口 per-token embedding(2048) + camera_id + scene_ctx]，label = navtrain 上 L12 head-avg attention 的 per-token ranking（复用现成 dump `exp/m1b2_navtrain_full_alllayers/<tok>.pt` 的 (28,16,720)），loss = LambdaRank。
- **诚实预期（先写下，防"自信没核"）**：scorer 是对 **attn_L12 的蒸馏**，其质量天花板 ≈ 老师 attn_L12(89.0@r=0.5)，**不可能达到 oracle 91.8**（oracle 用真 PDMS 选 r，蒸馏 attention 达不到；那道 gap 要靠 online GRPO/M2，本周期不做）。故**期望结果 = scorer 在 r=0.5 与 attn_L12 相差 ≈ ±0.5pt，且推理免 2-pass（cheap MLP 替代 pass1 attention 捕获）**。这本身是 M1c 的合法交付（"廉价蒸馏 scorer 在推理期匹配 attention selector，去掉 2-pass 开销"）。
- **明确不做（需 user 对齐范围）**：Budget Policy(4-class)、online GRPO（S3 重头，多天）。

**可行性核验（已读代码确认）**：现有 `patch_attention_capture`/`PromptIndex`/`vision_token_positions` 机制可直接扩展出一个 hook，在 LLM 第 0 层输入 hidden_states 上按 vision 位置 index_select → 得 per-token (720,2048) 特征 = scorer 输入。与现有 attention hook 同范式，低风险。

**Fallback / Bonus（若 Track B 受阻或提早完成）：把 S2 gate 从 shard0 加固到完整 4-shard navtest(n≈11574)**，消除 cond1 贴边(+0.49pt) 的 caveat。零新代码、复用 `run_tokenprune_S2_2gpu.sh`。HANDOFF 标为「可选加固，结论在 shard0 已稳」。
