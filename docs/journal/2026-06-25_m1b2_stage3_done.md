## 5. Errors (8 trajectory-pose AssertionErrors)

**Mechanism**: in `run_attention_probe.py`, attention `.pt` is saved **inside** the forward hook (`AttentionCapture.save()` is called during `model.forward()`). The trajectory-pose assertion fires **after** forward, during the PDMS sample preparation step that the probe inherits from the AutoVLAAgent base class. So:

- The 8 affected tokens **have their attention .pt saved** (this is why `.pt` count = 19,225, not 19,217).
- They are flagged as `err` in the probe summary because the trajectory metadata is malformed in the navtrain source data (well-known navsim noise: see `BSDPoseDecoder` consistency assertion in `navsim.common.dataclasses.Trajectory`).

**Affected tokens** (saved in `exp/m1b2_navtrain_full_alllayers/_stage3_trajectory_err_tokens.txt`):

```

ca8281be07935921
e97dbf85c52d56b9

```

**Downstream policy**:

- For **attention analysis / learned head-gating training** (M1.b₂ Phase 2): use all 19,225 .pt — trajectory-side noise is irrelevant for vision-attn modeling.
- For **navtrain PDMS evaluation** (future Phase F-style sweep on navtrain): drop these 8, work with 19,217 (matches the safe set already implicit in M0/M1.a). Sample mask file `_stage3_trajectory_err_tokens.txt` should be subtracted.
